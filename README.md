# VideoRecBack

VideoRecBack 是一个面向 NAS 视频存档的容器化浏览器。它把设置存到 `/config`，把 SQLite 数据库和 webp 缩略图缓存存到 `/data`，视频目录通过 Docker volume 只读挂载到容器内路径。

## 功能

- 时间线首页，只读取数据库和缓存图，不遍历视频目录，左侧按年份和季度显示 Immich 风格视频数量横条；可右键季度横条添加彩色标签。
- 首页使用置顶横向工具栏，时间线、文件夹、日历是三选一切换，筛选作为二级菜单打开。
- 首页支持按类型、时长、画幅筛选，并可用滑条调节封面预览大小；日历视图默认从年视图进入。
- 平面视频显示为 4:3 胶片卡片，封面是 4 个时间点合成的 1280x720 webp 四宫格。
- 全景视频显示为 4:3 卡片内的球面预览，封面优先取中点，失败后按多个时间点重试，并生成 960x720 webp 鱼眼球面预览。
- 点击封面直接播放，右键或长按封面进入该视频设置。
- 宽屏横屏下点击封面会在右侧展开内嵌播放器，并可拖动分隔条调整列表和播放器宽度比例；竖屏下点击封面跳转到播放页。
- 播放页保持固定视口高度，画质、倍速、曝光和音量使用对齐的紧凑滑条；画质可在原画、低压缩和高压缩之间切换，低压缩生成最高 1080P/5mbps HLS 分片串流，高压缩生成最高 720P/1mbps HLS 分片串流，切换时会从当前时间点边转码边续播并保持原视频长宽比；全景视频支持独立音量滑条、贴边环形进度拖动、拖动、触控板双指滑动、触屏双指缩放和触控板缩放来控制视角与视野，双击全景播放器可全屏。
- 界面固定为浅色主题。
- 设置页可调整页面标题、目标路径、按小时计的扫描间隔、默认播放音量、串流缓存保留和检查天数、视频扩展名、忽略点开头文件/目录、忽略文件名规则、日期/大小/时长显示项；扫描间隔为 0 时关闭自动全盘扫描，默认播放音量会同步写入数据库。
- 播放前不访问原视频文件；只有手动扫描、开启的自动扫描、增量文件/文件夹扫描和播放路由会读取视频目录。
- 扫描发现视频文件已删除时，会同步删除数据库中的视频记录、对应缩略图缓存和串流缓存。
- 播放接口支持 HTTP Range，适合浏览器拖动进度条。

## Docker Compose 模板

```yaml
services:
  videorecback:
    image: videorecback:0.4
    container_name: videorecback
    ports:
      - "8080:8080"
    volumes:
      - ./config:/config
      - ./data:/data
      - /path/to/nas/videos:/media:ro
    restart: unless-stopped
```

打开 `http://localhost:8080`，进入设置确认目标路径为 `/media`，然后点击扫描。

## 全景封面调试

容器或本地环境可运行 `python -m app.thumbnails /path/to/panorama360.mp4 /tmp/thumb.webp --duration 120`，输出实际取帧时间、封面尺寸、亮度/对比度、有效性和每次重试原因。

## 推荐用法

把本项目链接发送给你的 AI agent，并告诉它安装本项目、按你的 NAS 视频目录生成 Docker Compose 配置，然后启动容器。

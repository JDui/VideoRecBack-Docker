# VideoRecBack

VideoRecBack 是一个面向 NAS 视频存档的容器化浏览器。它把设置存到 `/config`，把 SQLite 数据库和 webp 缩略图缓存存到 `/data`，视频目录通过 Docker volume 只读挂载到容器内路径。

## 功能

- 时间线首页只读取数据库和缓存图，不遍历视频目录；视频卡片按 180 条分批加载，左侧时间线统计保持完整并支持跨批次跳转。
- 首页使用置顶横向工具栏，时间线、文件夹、日历是三选一切换，筛选作为二级菜单打开。
- 首页支持按类型、时长、画幅筛选，并可用滑条调节封面预览大小；日历视图默认从年视图进入。
- 平面视频显示为 4:3 胶片卡片，封面按设置的 480P/576P/720P 高度生成 4 个时间点合成的 webp 四宫格；10bit 视频会在封面右下角显示黄色 10bit 标记，历史记录会在下次播放入口懒检测一次并写入数据库。
- 全景视频显示为 4:3 卡片内的球面预览，封面优先取中点，失败后按多个时间点重试，并按设置的 480P/576P/720P 高度生成 webp 鱼眼球面预览；带全景标记或宽高接近 2:1 的视频会默认识别为全景，10bit 全景视频同样会显示 10bit 标记。
- 点击封面直接播放，右键或长按封面进入该视频设置。
- 播放页可收藏视频，首页右侧视图入口提供收藏页；收藏页中右键或长按视频可跳转到时间线位置或进入视频设置。
- 页面宽高比大于 4:3 时，点击封面会在右侧展开内嵌播放器，并可拖动分隔条调整列表和播放器宽度比例；宽高比不大于 4:3 时会跳转到完整播放页，关闭后会恢复原列表位置。
- 播放页保持固定视口高度，画质放在标题栏下拉菜单中，可在原画、超清、高清和流畅之间切换，并可在设置中分别选择平面/全景默认画质；原画直接走原始媒体路由推流，超清保持原分辨率并使用 8mbps HLS 分片串流，高清生成最高 1080P/3mbps HLS 分片串流，流畅生成最高 720P/1mbps HLS 分片串流，切换时会从当前时间点边转码边续播并保持原视频长宽比；10bit 原画在当前浏览器无法直接播放时会自动切到超清 HLS 转码保底播放；播放器会在缓冲卡住时重开当前画质并恢复播放，HLS 暂停和页面短暂隐藏不会立刻删除当前转码流；平面视频使用画面内覆盖式进度、音量和按钮控件，全景视频支持底部进度滑条、音量滑条、状态描边、拖动、触控板双指滑动、触屏双指缩放和触控板缩放来控制视角与视野，双击全景播放器可全屏。
- 界面固定为浅色主题。
- 设置页可调整页面标题、目标路径、按小时计的扫描间隔、默认播放音量、平面/全景默认画质、封面分辨率、平面/全景 HLS 转码编码器、HLS 最大缓存占用、内网直连、视频扩展名、忽略点开头文件/目录、忽略文件名规则、日期/大小/时长显示项，并可执行 2MB 样本的服务器连通测试；需要二次确认的刷新封面、全景重校验和重校验全部视频数据集中在同一个操作框内，其中全部视频数据重校验会重新记录 10bit、色度采样和平均码率；内网入口使用 JSON 和图片并行快速探测、短间隔重试及成功缓存，HTTPS 外网页面配置 HTTP 内网入口时会直接提供跳转按钮；扫描间隔为 0 时关闭自动全盘扫描，默认播放音量会同步写入数据库。
- 播放前不访问原视频文件；只有手动扫描、开启的自动扫描、增量文件/文件夹扫描和播放路由会读取视频目录。
- 全盘扫描使用文件时间纳秒、大小和路径索引批量比较，未变化文件不写数据库；媒体分析与封面生成进入单并发后台队列，并限制 ffmpeg 线程数以降低 CPU 峰值。
- 扫描发现视频文件已删除时，会同步删除数据库中的视频记录、对应缩略图缓存和串流缓存。
- 播放接口支持 HTTP Range，适合浏览器拖动进度条。

## Docker Compose 模板

```yaml
services:
  videorecback:
    image: videorecback:2.6.0
    container_name: videorecback
    ports:
      - "8080:8080"
    volumes:
      - ./config:/config
      - ./data:/data
      - /path/to/nas/videos:/media:ro
    restart: unless-stopped
```

打开 `http://localhost:8080`，进入设置确认目标路径为 `/media`，然后点击扫描。

## 全景封面调试

容器或本地环境可运行 `python -m app.thumbnails /path/to/panorama360.mp4 /tmp/thumb.webp --duration 120`，输出实际取帧时间、封面尺寸、亮度/对比度、有效性和每次重试原因。
